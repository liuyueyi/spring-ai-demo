# 博客创作技能

## 何时使用此技能
- 撰写技术教程和指南
- 分享项目实战经验
- 总结技术方案和架构设计
- 记录问题解决过程
- 编写产品使用文档
- 创建代码示例和最佳实践
- 撰写技术观点和分析文章

## 创作指导

### 第一步：明确博客定位和受众

**目标读者**：
- 初学者：需要基础概念讲解、详细步骤、大量示例
- 进阶开发者：关注实现细节、性能优化、最佳实践
- 资深专家：侧重架构设计、技术选型、深度分析

### 第二步：设计文章结构

**标准技术博客结构**：
```markdown
# 标题（清晰、具体、有吸引力）

## 引言
- 背景介绍
- 问题提出
- 文章价值
- 阅读预期

## 正文
### 1. 基础概念（可选）
### 2. 核心内容
### 3. 实践案例
### 4. 深入分析

## 总结
- 要点回顾
- 最佳实践建议
- 延伸思考

## 参考资料
```

### 第三步：写作技巧和规范

**标题撰写**：
- 清晰表达主题，避免标题党
- 包含关键词，便于搜索
- 突出价值点（如"完整指南"、"实战总结"、"深入解析"）
- 适当使用数字（如"5 个技巧"、"3 种方案"）

**内容组织**：
- 逻辑清晰，层次分明
- 由浅入深，循序渐进
- 重要内容前置
- 善用小标题分割长文

**代码示例**：
- 确保代码可运行
- 添加必要的注释
- 展示完整上下文
- 标注重点代码行
- 提供多种实现方案对比

**图表运用**：
- 架构图：展示系统整体结构
- 流程图：说明业务流程
- 时序图：描述交互过程
- 对比表：清晰展示差异
- 截图：辅助说明操作步骤

**语言表达**：
- 准确专业，避免歧义
- 通俗易懂，少用生僻术语
- 主动语态，增强可读性
- 适度幽默，增加趣味性

**实战总结示例**：
```markdown
# 一次生产环境 OOM 问题的排查与解决

## 问题背景
某日收到监控告警，应用频繁 Full GC，响应变慢...

## 排查过程
### 1. 查看监控指标
- heap 使用率持续 90%+
- Old 区只增不减

### 2. 导出堆 dump 分析
使用 jmap 导出：
```bash
jmap -dump:format=b,file=heap.hprof <pid>
```

### 3. MAT 分析
发现大量 XXX 对象未释放...

## 根本原因
代码中 XXX 处，缓存未设置过期时间...

## 解决方案
1. 添加缓存过期策略
2. 使用 LRU 淘汰算法
3. 限制缓存最大容量

## 经验教训
- 缓存必须设置过期时间
- 加强监控和告警
- 定期 review 缓存使用

## 注意事项

**避免的问题**：
1. 抄袭和侵权 - 坚持原创，引用需注明
2. 内容过时 - 及时更新或标注版本
3. 代码不可运行 - 务必测试验证
4. 错误引导 - 技术方案需经过实践检验
5. 广告嫌疑 - 客观评价，避免过度营销

**最佳实践**：
1. 保持持续输出
2. 建立个人品牌
3. 与读者互动反馈
4. 定期回顾和优化旧文
5. 形成系列专题
